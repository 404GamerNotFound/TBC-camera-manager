Externe Quellen

Externe Quellen installieren Plugins direkt aus einem öffentlichen GitHub-Repository, statt eine ZIP-Datei manuell hochzuladen. Ein Sync lädt und installiert den aktuellen Stand erneut; nur öffentliche Repositories werden unterstützt. Jede registrierte Quelle wird zusätzlich stündlich automatisch auf ein neueres Update geprüft - siehe Admin → Updates.

Erforderliche Plugin-Struktur

Ein Repository (oder der angegebene Unterordner darin) muss dieselbe Struktur wie ein manueller ZIP-Import haben:

Kamera-Plugin
manifest.json, plugin.py, module.py, optional detections.json und tests/test_*.py
Cloud-Anbieter
manifest.json, plugin.py, module.py, optional tests/test_*.py
Design
manifest.json, static/styles.css - kein Code, keine Tests

manifest.json muss schema_version, key, label, version und (außer bei Designs) entrypoint enthalten; plugin.py stellt create_module() bereit, das eine CameraModule- bzw. CloudAccountModule-Instanz zurückgibt. Ein optionaler tests/-Ordner mit pytest-Tests kann direkt in TBC über „Tests ausführen“ gestartet werden. Die vollständige Referenz steht in docs/plugin-sources.md im TBC-Repository sowie docs/camera-modules.md/docs/cloud-accounts.md/docs/design-themes.md.

External sources

External sources install plugins directly from a public GitHub repository instead of requiring a manual ZIP upload. Synchronizing downloads and installs the latest version again; only public repositories are supported. Each registered source is also checked hourly for a newer update; see Admin → Updates.

Required plugin structure

A repository (or the specified subdirectory within it) must have the same structure as a manual ZIP import:

Camera plugin
manifest.json, plugin.py, module.py, optional detections.json and tests/test_*.py
Cloud providers
manifest.json, plugin.py, module.py, optional tests/test_*.py
Network providers
manifest.json, plugin.py, module.py, optional tests/test_*.py
Design
manifest.json, static/styles.css - no code, no tests

manifest.json must contain schema_version, key, label, version and, except for designs, entrypoint. plugin.py provides create_module(), which returns a CameraModule or CloudAccountModule instance. An optional tests/ folder containing pytest tests can be run directly in TBC using “Run tests”. The full reference is available in docs/plugin-sources.md in the TBC repository, as well as docs/camera-modules.md, docs/cloud-accounts.md and docs/design-themes.md.

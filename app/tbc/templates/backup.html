Backup & restore

Create backup

Downloads an encrypted archive of the database (cameras, users, settings, cloud accounts, API tokens). Stored credentials stay encrypted with this instance's TBC_SECRET_KEY - keep that key safe, it's required to restore.

Restore from backup

Restoring replaces the current database with the one in the uploaded archive. The current database is copied to a .bak file first. The archive must have been created with the same TBC_SECRET_KEY this instance is running with now. Restart the app after restoring.
